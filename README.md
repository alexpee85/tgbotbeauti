# Описание бота для мастера (салона) по маникюру и другим услугам сферы бьюти

Telegram-бот для записи клиентов на приём: выбор даты и времени, услуги из прайса, проверка подписки на канал, уведомления администратору и напоминания клиентам.

## Реализованные функции

### Для клиента

| Функция | Описание |
| :--- | :--- |
| **Запись на приём** | Клиент выбирает дату (календарь на месяц вперёд), свободное время и услугу из прайса, вводит имя и телефон, подтверждает запись. Один пользователь может иметь только одну активную запись. |
| **Проверка подписки на канал** | Перед записью бот проверяет подписку на указанный канал. Без подписки запись недоступна; есть кнопки «Подписаться» и «Проверить подписку». При желании клиент может вернуться в главное меню. |
| **Прайс-лист** | Кнопка «Прайсы» показывает актуальный список услуг и цен из базы (формируется администратором). Под сообщением — кнопка «В главное меню». |
| **Портфолио** | Кнопка «Портфолио» открывает ссылку на работы (например, Pinterest) и кнопку возврата в главное меню. |
| **Отмена своей записи** | Клиент может отменить активную запись через кнопку «Отменить запись»: подтверждение — и слот снова становится доступным для других. Администратор получает уведомление об отмене (имя, телефон, услуга, дата/время). |
| **Напоминание за 24 часа** | За сутки до визита клиенту автоматически отправляется напоминание (дата, время, услуга). Если запись создана менее чем за 24 часа до приёма, напоминание не создаётся. При отмене записи напоминание отменяется. |
| **Возврат в меню** | В ключевых шагах (прайсы, портфолио, проверка подписки, ввод имени/телефона при записи) есть кнопка «В главное меню» — не нужно каждый раз вводить /start. |

### Для администратора

| Функция | Описание |
| :--- | :--- |
| **Доступ по ID** | Админ-панель доступна только пользователям, чьи Telegram ID указаны в настройках (ADMIN_IDS). |
| **Рабочие дни** | Добавление рабочих дней в формате ДД.ММ.ГГГГ. Отображаемый календарь для клиентов строится по этим дням. |
| **Временные слоты** | Добавление и удаление слотов (например, 10:00, 11:00). Список слотов общий для всех рабочих дней; при записи клиента слот на выбранную дату становится занятым. |
| **Закрытие дня** | Возможность полностью закрыть выбранный день — на эту дату нельзя будет записаться. |
| **Расписание на дату** | Просмотр всех записей на выбранную дату: время, имя, телефон, услуга. Видно, закрыт ли день. |
| **Отмена записи клиента** | Список предстоящих записей с возможностью отменить любую (например, по просьбе клиента). После отмены слот освобождается, напоминание не отправляется. |
| **Управление прайсом** | Раздел «Прайсы»: просмотр списка услуг; добавление новой услуги (название + цена); удаление услуги. Изменения сразу видны клиентам в кнопке «Прайсы» и в выборе услуги при записи. |

### Уведомления и каналы

| Функция | Описание |
| :--- | :--- |
| **Уведомление админу о новой записи** | В личные сообщения каждому администратору приходит сообщение: имя, телефон, выбранная услуга, дата и время, ID пользователя. |
| **Уведомление админу об отмене** | При отмене записи клиентом администратор получает сообщение: кто отменил (имя, телефон, услуга), дата и время записи, ID пользователя и ID записи. |
| **Публикация в канал расписания** | При подтверждении записи бот отправляет сообщение в указанный канал (дата, время, имя клиента, при необходимости — услуга). Удобно для общего расписания или для второго мастера. |

### Технические особенности

| Аспект | Описание |
| :--- | :--- |
| **Интерфейс** | Везде используются inline-кнопки: календарь, выбор времени и услуги, подтверждение, возврат в меню, админ-панель. |
| **Состояния (FSM)** | Сценарии записи и админки реализованы через конечный автомат (FSM): нельзя «потеряться» в диалоге; при необходимости можно вернуться в меню. |
| **База данных** | SQLite: одна база на сервере, не требует отдельного сервера СУБД. Хранятся рабочие дни, слоты, записи (включая выбранную услугу), задачи напоминаний. |
| **Напоминания после перезапуска** | Задачи напоминаний за 24 часа сохраняются в БД. При перезапуске бота (обновление, перезагрузка сервера) запланированные напоминания восстанавливаются. |
| **Форматирование** | Сообщения оформлены в HTML (жирный текст, переносы строк) для удобного чтения в Telegram. |

## Преимущества использования бота

1. **Экономия времени** — клиенты сами выбирают дату, время и услугу; не нужно согласовывать по телефону или в переписке.
2. **Актуальный прайс** — цены и список услуг задаются в админке и сразу отображаются клиентам; не нужно обновлять посты или картинки вручную.
3. **Контроль доступа** — обязательная подписка на канал перед записью помогает увеличить охват канала и информировать клиентов.
4. **Меньше пропусков** — напоминание за 24 часа снижает количество «неявок» и пустых слотов.
5. **Прозрачность** — администратор видит все записи и отмены в одном месте; при необходимости можно отменить запись или закрыть день.
6. **Один слот на клиента** — нельзя случайно записаться на несколько дат; при отмене слот сразу снова доступен другим.
7. **Удобство для клиента** — всё в одном чате с ботом: прайс, портфолио, запись и отмена без ввода /start на каждом шаге.
8. **Готовность к работе на VPS** — бот можно развернуть на недорогом сервере (1 ГБ RAM, 1 ядро) и держать включённым 24/7 с автоперезапуском при сбоях.

## Итог

Бот объединяет запись на приём, выбор услуги из прайса, проверку подписки, уведомления администратору и клиентам и управление расписанием через админ-панель. Подходит для мастера по маникюру (или другого специалиста с почасовой записью), который хочет вести записи и прайс в Telegram без лишних сервисов и звонков.
